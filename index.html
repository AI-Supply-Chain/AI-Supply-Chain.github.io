User Manual

1. Overview

The simulator performs targeted traversals on a large, precomputed AI model network stored in Graphviz .dot format. Two traversal modes are supported: Forward/Downstream Traversal for downstream derivations and Backward/Upstream Traversal for upstream provenance.

2. Quick Start

  1. Click Launch Simulation to open the interactive app.
  2. Ensure the Graph File Path points to your served .dot file (default: AI-Supply-Chain-Graph.dot).
  3. Choose a direction: Forward/Downstream or Backward/Upstream.
  4. Type 2â€“3 characters in the model search box, select a model, then click Run Traversal.
  5. Explore the visualization and copy/download the generated DOT subgraph.

3. Legend & Semantics

  • FT â€” Fine-tuned
  • AD â€” Adapter (LoRA/QLoRA)
  • QN â€” Quantization (incl. GGUF)
  • MR â€” Merged models
  • Node color encodes depth (level).
  • Red outline marks extremes: terminal nodes in forward mode, base models in backward mode.
  • Edge labels show the abbreviations above.

4. Data Requirements

Input must be a valid Graphviz digraph. Edges can provide label= attributes (e.g., label="finetune") to classify as FT/AD/QN/MR; otherwise a sensible default (FT) is applied.

5. Exported DOT

The app emits a compact DOT with level ranks, edge-type labels, and comments detailing node counts, max depth, and paths to extremes (terminal/base).

6. Performance Tips

  • Large subgraphs automatically use Sigma/Graphology for fast rendering.
  • Use Desired Graph Level to cap traversal depth for responsiveness.
  • Edge labels are drawn for small graphs in SVG and for big graphs via Sigma edge labels.
